# 公告驅動基本面事件策略

## 核心假設

我們不是等券商報告、新聞整理或二手資料，而是直接使用公開資訊觀測站公告時間點。
公告提供「市場正式知道的時間」，本地資料庫提供「這個事件是否值得交易」。

## 資料層

1. 公告事件資料庫
   - 來源：`D:\dataset\processed\material_events.jsonl`
   - 欄位：公告日期、公告時間、股票代號、公司名稱、公告主旨、公告內容、事件類型。
   - 用途：決定何時可以買。

2. 基本面特徵資料庫
   - 來源：公告內財務業務資訊，以及未來補上的月營收、財報資料。
   - 欄位：營收、營收年增率、稅前淨利、EPS、EPS 年增率、毛利率、營益率、淨利率。
   - 用途：做三率三升、營收動能、轉虧為盈等條件過濾。

3. 股價資料庫
   - 來源：`D:\dataset\processed\prices`
   - 欄位：開高低收、成交量、成交值。
   - 用途：隔日買入、月底或隔月第 N 個交易日賣出。

4. 法人資料庫
   - 目前 `D:\dataset` 尚未看到外資、投信、自營商處理後資料。
   - 先保留欄位與缺口報告，等資料補上後接入。

## 初版交易規則

1. 事件範圍：2026-01-01 到 2026-06-30 的財務/營收公告。
2. 進場：公告後下一個交易日開盤買入。
3. 出場：參數化。
   - `month_end`：進場月份月底最後一個可交易日收盤賣出。
   - `next_month_n`：隔月第 N 個交易日收盤賣出。
4. 資金：每月 100 萬。
5. 選股：每月依分數排序取 Top N，資金等權配置。

## 初版可掃參數

- `top_n`: 5, 10
- `min_revenue_yoy_pct`: 0%, 10%, 20%, 50%
- `min_eps_yoy_pct`: 0%, 20%, 50%
- `require_positive_eps`: true, false
- `require_three_rises`: true, false
- `exit_rule`: month_end, next_month_3, next_month_5, next_month_10

## 三率三升定義

初版先拆成兩組，避免資料還沒補齊時硬湊。

1. 三升
   - 營收年增率 > 0
   - 稅前淨利年增率 > 0
   - EPS 年增率 > 0

2. 三率
   - 毛利率
   - 營業利益率
   - 淨利率

目前公告第 51 款的注意交易財務資訊常有營收、稅前淨利、EPS，但不一定有毛利率與營益率。
因此三率欄位先保留，等月營收/財報資料庫補齊後納入回測條件。
