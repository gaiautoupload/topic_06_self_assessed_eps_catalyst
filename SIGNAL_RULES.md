# Signal Rules Draft

## Signal Goal

把公告訊息分成三個層級，讓回測與網頁展示都可以共用。

## Signal Strength

### A

符合以下條件：

- 標題明確出現 `自結` 或 `自結 EPS`
- 內容可直接抽出 EPS
- 事件屬於公司正式公告或法說會正式文件

### B

符合以下條件：

- 內容明確出現自結獲利
- EPS 沒有直接出現，但可由獲利與股本推算
- 事件仍為公告性質

### C

符合以下條件：

- 有自結語境，但內容不完整
- 只有部分數字，無法穩定抽取 EPS
- 可先保留做觀察，不進主策略

## Keyword Set

### Positive Keywords

- `自結`
- `EPS`
- `每股盈餘`
- `稅後純益`
- `獲利`
- `盈餘`
- `財報`
- `法說`

### Negative Keywords

- `新聞`
- `評論`
- `分析`
- `預估`
- `法人看法`
- `目標價`

## Extraction Priority

1. 先看標題
2. 再看正文第一段
3. 再看全文數字
4. 找不到 EPS 時，改抓稅後純益並嘗試推算

## Dedup Rules

- 同公司同日同期間只留一筆主事件
- 修正版公告優先保留資訊最完整者
- 標題相近且內容高度重疊視為同事件
